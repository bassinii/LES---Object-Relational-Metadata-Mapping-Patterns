Metadata Mapping

Guarda detalhes de mapeamento objeto-relacional em metadados.

O código que lida com o mapeamento objeto-relacional descreve como os campos no banco de dados correspondem a campos em objetos.

  • Enfadonho
  • Repetitivo. Um mapeamento de metadados permite aos programadores definir os mapeamentos de uma forma simples, que então é processado pelo código genérico para realizar a leitura, inserção, e a atualização dos dados.

Como Funciona:

A maior decisão em usar Metadados Mapeamento é a forma como as informações nos metadados se manifesta em termos de execução de código. Existem duas principais rotas a tomar:

  • Geração de código;
  • Programação reflexiva;

Geração de Código:

A aplicação recebe como entrada metadados e cuja saída é o código-fonte das classes que fazem o mapeamento. As classes geradas nunca devem ser editadas manualmente. Menos dinâmico:

  • Quaisquer alterações no mapeamento requerem recompilação e redistribuição.

Programação Reflexiva:

Trata os métodos (e campos) como dados. O programa reflexivo pode ler o nome do campo ou do método em um arquivo de metadados e usá-los para realizar o mapeamento. É desencorajado de se usar programação reflexiva pois:

  • Lento;
  • Difícil depuração Porem torna a aplicação mais flexível.

Na maioria das vezes você mantenha os metadados em um formato de arquivo separado. Atualmente XML é uma escolha popular, pois proporciona estruturação hierárquica, libertando-o de escrever seus próprios analisadores e outras ferramentas. Um dos desafios dos metadados é que existem casos onde terá que ser adicionado uma grande complexidade de metadados, uma alternativa útil é sobrescrever o código genérico .

Quando Usá-lo:

Esse padrão reduz muito a quantidade de trabalho para tratar o mapeamento em um banco de dados, mas o trabalho de configuração é requerido para preparar o framework. O mapeamento em metadados pode interferir na refatoração, mas por outro lado pode tornar a refatoração no banco de dados mais fácil, já que os metadados representam uma declaração na interface.
